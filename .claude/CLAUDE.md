# Interview Prep Blog — Claude Code Context

## Project
Next.js 14, App Router, SSG, GitHub Pages deployment.
src/ directory. TypeScript strict mode. Tailwind CSS.

## Commit Message Rules
- Format: type: short description
- Types: feat, fix, post, refactor, style, chore, docs
- NEVER include "Co-authored-by", "Generated-by", or any AI attribution
- NEVER mention Claude, AI, or automated tools in commit messages
- Example: "feat: add TOC scroll highlighting"
- Example: "post: add react-hooks-deep-dive article"

## Code Style
- Tabs for indentation (not spaces)
- Single quotes for strings
- No semicolons (unless required)
- TypeScript strict: true

## Architecture Rules
- All data fetching in lib/ — never in components
- Server Components by default — "use client" only when needed
- generateStaticParams() required on all dynamic routes
- No API routes (static export only)
- Images must use unoptimized: true

## File Naming
- Components: PascalCase.tsx
- Lib files: camelCase.ts
- Content: kebab-case.md
- Types: PascalCase in types/blog.ts

## DO NOT
- Add console.log to production code
- Use any type (use unknown or proper types)
- Create API routes (breaks static export)
- Use next/image without unoptimized flag
- Write "Generated by Claude" or similar anywhere
- Add Co-authored-by trailers to git commits
